基础#

默认文件格式#

除了使用 Sphinx 的默认格式，RestructuredText （.rst），加载 myst_nb 也会解析：

  • Markdown 文件 （.md）

  • Jupyter 笔记本（.ipynb）

  • 基于文本的笔记本 MyST-NB（.md + top-matter）

自定义文件扩展名#

你可以使用 conf.py 中的 source_suffix 选项来改变 MyST-NB 解析的文件扩展名，例如：

extensions = ["myst_nb"]
source_suffix = {
    '.rst': 'restructuredtext',
    '.ipynb': 'myst-nb',
    '.myst': 'myst-nb',
}

其他笔记本格式#

请参阅 Custom Formats 节，了解如何将其他笔记本格式集成到您的构建中，以及如何与 jupytext 集成。

MyST Markdown#

对于所有的文件格式，Markdown 创作是 MyST-NB 的支柱。默认情况下，MyST 风格的 Markdown 是启用的，它扩展了 CommonMark，使用 RST 启发的语法，提供技术写作所需的额外功能。

特别是 MyST 添加了目标（targets）、角色（roles）和指令（directives）语法，允许你利用所有可用的 Docutils/Sphinx 特性：

RestructuredText
.. _target:
Header
------

:role-name:`content`

.. directive-name:: argument
   :parameter: value

   content
MyST Markdown
(target)=
# Header

{role-name}`content`

```{directive-name} argument
:parameter: value

content
```

参见 Jupyter 笔记本 部分，了解如何编写 Jupyter 笔记本的更多细节。

基于文本的笔记本#

MyST-NB 文本笔记本是用于在文本文件中存储 Jupyter 笔记本的特殊格式。它们直接映射到 Notebook 文件，而不直接存储代码执行输出。

要将 Markdown 文件指定为基于文本的记事本，请在文件开头添加以下 top-matter：

---
file_format: mystnb
kernelspec:
  name: python3
---

kernelspec.name 应该与安装在您的环境中的 Jupyter 内核 相关。

MyST-NB 也将认可 jupytext 的顶级内容，比如：

---
kernelspec:
  name: python3
  display_name: python3
jupytext:
  text_representation:
    extension: .md
    format_name: myst
    format_version: '0.13'
    jupytext_version: 1.13.8
---

然后，代码单元格由 code-cell 指令指定：

```{code-cell}
:tags: [my-tag]

print("Hello world!")
```

而且 Markdown 可以通过 +++ break 语法拆分为单元格：

Markdown cell 1

+++ {"tags": ["my-tag"]}

Markdown cell 2, with metadata tags

请参阅 基于文本的笔记本 部分，以获得关于基于文本的笔记本以及与 jupytext 集成的更多详细信息。

配置#

MyST-NB 的解析、执行和呈现可以在三个层次的特异性上配置；在全局范围内，每个文件和每个笔记本单元，最具体的配置优先。

关于如何配置 MyST-NB 的更多细节，请参见 Configuration 部分。
