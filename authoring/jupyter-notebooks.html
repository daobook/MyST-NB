Jupyter 笔记本#

This notebook is a demonstration of directly-parsing Jupyter Notebooks into Sphinx using the MyST parser.1

Markdown#

参见

关于 MyST Markdown 的更多信息，请参见 MyST 解析器语法指南。

配置#

MyST-NB 解析器源自 基本的 MyST-Parser，所以所有的配置选项都是可用的。请参阅 MyST 配置选项 以获得完整的选项集，以及 MyST 语法指南 以获得所有语法选项。

要从这个笔记本构建文档，需要设置以下选项：

myst_enable_extensions = [
    "amsmath",
    "colon_fence",
    "deflist",
    "dollarmath",
    "html_image",
]
myst_url_schemes = ("http", "https", "mailto")

备注

加载 myst_nb 扩展也会激活 myst_parser 扩展，以启用 MyST 的“Markdown”风格。不需要在 extensions 列表中显式地添加它。

语法#

正如您所看到的，markdown 是按照预期进行解析的。嵌入图像应该可以正常工作。例如，这是 MyST-NB 的 logo：

![myst-nb logo](../_static/logo-wide.svg)

通过在 sphinx 配置中的 myst_enable_extensions 列表中添加 "html_image" （参见这里），你甚至可以添加带有属性的 HTML img tag：

<img src="../_static/logo-wide.svg" alt="logo" width="200px" class="shadow mb-2">

因为 MyST-NB 使用了 MyST-markdown 解析器，你可以在你的笔记本中加入 Sphinx 的丰富的 markdown 功能。例如，这里有注释警告块：

备注

Wow, a note! It was generated with this code (as explained here):

:::{note}
**Wow**, a note!
:::

如果您希望使用裸 LaTeX 方程，那么应该将 "amsmath" 添加到 sphinx 配置中的 myst_enable_extensions 列表中。这里 解释了它 的工作原理：

\begin{equation}
\frac {\partial u}{\partial x} + \frac{\partial v}{\partial y} = - \, \frac{\partial w}{\partial z}
\end{equation}

\begin{align*}
2x - 5y &=  8 \\
3x + 9y &=  -12
\end{align*}
(1)#\[\begin{equation} \frac {\partial u}{\partial x} + \frac{\partial v}{\partial y} = - \, \frac{\partial w}{\partial z} \end{equation}\]
\[\begin{align*} 2x - 5y &= 8 \\ 3x + 9y &= -12 \end{align*}\]

你也可以使用诸如公式编号并在笔记本上引用的功能。

$$e^{i\pi} + 1 = 0$$ (euler)
(2)#\[e^{i\pi} + 1 = 0\]

欧拉等式，(2)，被选为最美丽的数学公式之一。

你可以在 MyST 文档中看到 这个例子所使用的语法。

Raw 单元格#

原始单元类型 可用于将内容特定地呈现为特定的 MIME 媒体类型。

```{raw-cell}
:format: text/html

<p>My cat is <strong>very</strong> grumpy.</p>
```

My cat is very grumpy.
